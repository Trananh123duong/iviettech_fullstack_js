Bảng danh sách câu hỏi phỏng vấn

Kỹ năngCâu hỏiTrả lời
HTMLHTML5 là gì? Nó khác gì so với các phiên bản trước?HTML5 là phiên bản mới nhất của HTML, cung cấp nhiều tính năng mới như:
• Các thẻ
semantic mới (<header>, <footer>, <article>, <section>, ...).
• Hỗ trợ tốt hơn cho
đa phương tiện với <audio> và <video>.
• Các API mới như
Canvas API, Local Storage, WebSockets.
• Hỗ trợ tốt hơn cho thiết bị di động.
HTMLHTML có phải là một ngôn ngữ lập trình không?• Không! HTML chỉ là một ngôn ngữ đánh dấu, không có logic điều kiện, vòng lặp hay biến như các ngôn ngữ lập trình thực thụ.
HTMLThẻ <meta> trong HTML dùng để làm gì?• Thẻ <meta> cung cấp thông tin về dữ liệu (metadata) của trang web, chẳng hạn như mô tả, từ khóa, tác giả, và các thông tin khác. Thẻ này thường được đặt trong phần <head> của tài liệu HTML và không hiển thị trực tiếp trên trang web.
HTMLSemantic HTML là gì? Tại sao nên dùng?• Semantic HTML: dùng các thẻ có ý nghĩa rõ ràng (<header>, <article>, <footer>) giúp SEO tốt hơn, accessibility dễ hơn, và code dễ đọc hơn.
HTMLCác cách để tối ưu SEO trong HTML?• Dùng thẻ meta (<meta name="description" content="...">).
• Đặt từ khoá trong heading (
<h1>, <h2>).
• Dùng
alt cho thẻ <img> để Google hiểu nội dung ảnh.
• Sử dụng structured data (Schema.org).
HTMLSự khác nhau giữa thẻ <div> và <span>?• <div> là block-level element, chiếm toàn bộ chiều ngang.
•
<span> là inline element, chỉ chiếm nội dung của nó.
HTMLSự khác biệt giữa thẻ <b> và <strong>, <i> và <em> là gì?• Thẻ <b> và <i> được sử dụng để làm đậm và in nghiêng văn bản mà không có ý nghĩa ngữ nghĩa đặc biệt.
• Thẻ
<strong> và <em> cũng làm đậm và in nghiêng văn bản, nhưng chúng mang ý nghĩa ngữ nghĩa, nhấn mạnh tầm quan trọng hoặc sự nhấn mạnh của văn bản.
HTMLCác thuộc tính quan trọng của thẻ <a> là gì?• href: Đường dẫn của liên kết.
•
target: _self (mặc định, mở trong tab hiện tại), _blank (mở tab mới).
•
rel: nofollow (không cho Google bot theo dõi liên kết).
HTMLThẻ <figure> và <figcaption> dùng làm gì?• <figure> chứa hình ảnh, biểu đồ hoặc nội dung minh họa.
•
<figcaption> là chú thích cho nội dung đó.
HTMLKhác biệt giữa id và class?• id: duy nhất trong trang, dùng khi cần chọn phần tử cụ thể.
•
class: có thể dùng cho nhiều phần tử, dùng khi cần nhóm nhiều phần tử lại.
HTMLIframe là gì và nó hoạt động như thế nào?• Thẻ <iframe> được sử dụng để nhúng một trang HTML khác vào trong trang hiện tại. Nó cho phép hiển thị nội dung từ một nguồn khác trong một khung trên trang web.
HTMLLazy loading là gì? HTML5 có hỗ trợ không?• Lazy loading giúp trì hoãn tải hình ảnh cho đến khi cần thiết, tiết kiệm băng thông. HTML5 hỗ trợ với thuộc tính loading="lazy".
HTMLCORS là gì? HTML có vai trò gì trong việc xử lý CORS?• CORS (Cross-Origin Resource Sharing): cơ chế cho phép resource từ domain khác được truy cập.
• HTML không trực tiếp xử lý CORS, nhưng thông qua thẻ
<script>, <img>, hoặc <iframe> có thể yêu cầu resource từ domain khác.
HTMLKhi nào nên dùng thuộc tính defer và async với thẻ <script>?• defer: chạy script sau khi HTML load xong (thích hợp khi script phụ thuộc vào DOM).
•
async: chạy script ngay khi tải xong, không đợi HTML (thích hợp cho các script không phụ thuộc DOM, như analytics).
HTMLTại sao nên dùng thuộc tính lang trong thẻ <html>?Thuộc tính lang xác định ngôn ngữ của tài liệu, giúp:
• Công cụ tìm kiếm (SEO) hiểu ngôn ngữ chính của trang.
• Trình đọc màn hình đọc đúng ngôn ngữ.
• Dịch tự động của trình duyệt hoạt động chính xác hơn.
HTMLCookies, Session Storage và Local Storage khác nhau như thế nào?• Cookies: Dùng để lưu dữ liệu nhỏ, thường gửi lên server với mỗi request, thời hạn do lập trình viên thiết lập.
•
Session Storage: Lưu dữ liệu trong phiên làm việc (tab), mất khi đóng tab.
•
Local Storage: Lưu dữ liệu lâu dài (trừ khi bị xóa thủ công), dùng cho thông tin cần giữ lại giữa các lần truy cập.
HTMLData attributes là gì và sử dụng như thế nào?• Data attributes cho phép lưu trữ thông tin tùy chỉnh trên các phần tử HTML với cú pháp data-*. Chúng thường được dùng trong JavaScript để thao tác dữ liệu.
CSSCó những cách nào để thêm CSS vào trang web?Có ba cách để thêm CSS vào trang web:
•
Inline CSS: Sử dụng thuộc tính style trực tiếp trong thẻ HTML.
•
Internal CSS: Đặt mã CSS trong thẻ <style> bên trong phần <head> của tài liệu HTML.
•
External CSS: Liên kết tệp CSS bên ngoài bằng thẻ <link> trong phần <head>.
CSSSự khác biệt giữa class selector và ID selector là gì?• Class selector (.classname) được sử dụng để áp dụng kiểu dáng cho một hoặc nhiều phần tử.
• ID selector (
#idname) được sử dụng để áp dụng kiểu dáng cho một phần tử duy nhất và phải là duy nhất trong tài liệu.
CSSBox Model là gì? Gồm những phần nào?• Box Model gồm: Content (nội dung), Padding (khoảng trống xung quanh content), Border (viền), Margin (khoảng cách bên ngoài).
CSSFlexbox và Grid khác nhau như thế nào? Khi nào nên dùng cái nào?• Flexbox: thiết kế theo một chiều (row hoặc column), cho phép sắp xếp các phần tử con trong một container theo hàng hoặc cột, với khả năng co giãn và căn chỉnh linh hoạt. Nên sử dụng Flexbox khi cần bố trí các phần tử trong một chiều (hàng hoặc cột) với sự linh hoạt cao.
•
Grid: thiết kế theo cả hai chiều (row và column), cung cấp khả năng kiểm soát chính xác vị trí và kích thước của các phần tử trong lưới. Nên sử dụng Gird cho layout phức tạp, chia ô.
CSSResponsive Design là gì? Các cách để làm responsive?• Responsive Design: thiết kế giao diện phù hợp với nhiều kích thước màn hình.
• Các cách làm responsive:
◦
Media queries (@media screen and (max-width: 768px) {...}).
◦
Flexible grid layout (dùng %, vh, vw thay vì px).
◦
Responsive images (<img srcset="...">).
CSSPseudo-elements và Pseudo-classes khác nhau thế nào?• Pseudo-classes: chọn trạng thái cụ thể của element (:hover, :focus).
•
Pseudo-elements: tạo và style một phần của element (::before, ::after).
CSSSự khác biệt giữa thuộc tính position: relative; và position: absolute; là gì?• position: relative; định vị phần tử tương đối so với vị trí ban đầu của nó trong luồng tài liệu.
•
position: absolute; định vị phần tử tương đối so với phần tử cha gần nhất có thuộc tính position khác static, hoặc so với viewport nếu không có phần tử cha như vậy.
CSSZ-index là gì và nó hoạt động như thế nào?• z-index là thuộc tính xác định thứ tự xếp chồng của các phần tử được định vị (có position khác static). Phần tử có z-index cao hơn sẽ được hiển thị trên phần tử có z-index thấp hơn.
CSSSự khác biệt giữa display: none; và visibility: hidden; là gì?• display: none; ẩn phần tử và loại bỏ nó khỏi luồng tài liệu.
•
visibility: hidden; ẩn phần tử nhưng vẫn giữ không gian của nó trong bố cục.
CSSKhác biệt giữa em, rem, px, và % trong CSS?• px: Đơn vị cố định, không phụ thuộc vào phần tử cha.
•
em: Phụ thuộc vào kích thước font của phần tử cha. Ví dụ, nếu font-size của cha là 16px, thì 1em = 16px.
•
rem: Phụ thuộc vào font-size của thẻ <html>, thường mặc định là 16px.
•
%: Phụ thuộc vào phần tử cha, dùng chủ yếu để xác định kích thước tương đối.
CSSLàm sao để ẩn một phần tử nhưng vẫn giữ không gian của nó?• Dùng visibility: hidden; thay vì display: none;, vì display: none; sẽ loại bỏ phần tử khỏi luồng tài liệu
JSJavaScript là ngôn ngữ đơn luồng (single-threaded), vậy nó xử lý bất đồng bộ (asynchronous) như thế nào?• JS dùng Event Loop và Callback Queue để xử lý bất đồng bộ. Khi gặp async task (setTimeout, API call, event listener...), nó đưa vào Web APIs, sau đó callback sẽ được đẩy vào hàng đợi và xử lý khi call stack rảnh.
JSGiải thích var, let, const, sự khác biệt giữa chúng?• var: Có phạm vi function-scoped, có thể khai báo lại, có hoisting lên top với giá trị undefined nhưng không có block-scoped.
•
let: Có phạm vi block-scoped, không thể khai báo lại trong cùng một phạm vi, có hoisting nhưng không khởi tạo.
•
const: Giống let, nhưng giá trị không thể thay đổi sau khi gán ban đầu.
JSHoisting là gì? Nó ảnh hưởng đến biến và function như thế nào?• Hoisting là cơ chế JS đưa khai báo biến và function lên đầu scope trước khi chạy code.
•
var hoisting nhưng gán undefined, let/const hoisting nhưng không thể dùng trước khi khai báo.
• Function declaration hoisting đầy đủ (gọi trước khai báo được), còn function expression thì không.
JSClosure trong JavaScript là gì?• Closure là một function có thể nhớ và truy cập các biến trong phạm vi cha của nó ngay cả khi function cha đã thực thi xong.
JSSự khác biệt giữa null và undefined?• null: Giá trị rỗng do lập trình viên gán.
•
undefined: Biến chưa được gán giá trị.
JSCall, Apply, Bind trong JavaScript là gì?• call(): Gọi hàm với this và tham số riêng lẻ.
•
apply(): Giống call, nhưng truyền tham số dưới dạng mảng.
•
bind(): Trả về một hàm mới với this được gán cố định.
JSSự khác biệt giữa map(), forEach(), filter() và reduce()?• map(): Tạo mảng mới với giá trị được biến đổi.
•
forEach(): Lặp qua từng phần tử nhưng không tạo mảng mới.
•
filter(): Trả về mảng mới chỉ chứa các phần tử thỏa điều kiện.
•
reduce(): Gộp các phần tử thành một giá trị duy nhất.
JSSự khác biệt giữa setTimeout và setInterval?• setTimeout(): Chạy một lần sau khoảng thời gian chỉ định.
•
setInterval(): Chạy lặp lại sau mỗi khoảng thời gian.
JSCallback Hell là gì? Làm sao để tránh nó?• Callback Hell là tình trạng code lồng nhau quá nhiều callback, gây khó đọc.
• Giải pháp: Dùng
Promises, Async/Await để viết code rõ ràng hơn.
JSEvent Loop hoạt động như thế nào?• Event Loop là cơ chế giúp JavaScript xử lý bất đồng bộ dù chỉ chạy trên một luồng duy nhất (Single Thread). Nó hoạt động bằng cách:
1.
Call Stack: Chạy các tác vụ đồng bộ trước.
2.
Web APIs / Background Tasks: Xử lý tác vụ bất đồng bộ như setTimeout, fetch.
3.
Callback Queue / Microtask Queue: Khi tác vụ bất đồng bộ hoàn thành, callback của nó được đẩy vào hàng đợi.
4.
Event Loop: Kiểm tra nếu Call Stack rảnh thì lấy callback từ hàng đợi để thực thi.
JSPrototype trong JavaScript là gì?• Prototype là cơ chế giúp các object trong JS kế thừa các thuộc tính và phương thức từ một object khác.
JSthis hoạt động như thế nào?• Global scope: this là window (hoặc global trong NodeJS).
• Object method:
this là object gọi method.
• Arrow function:
this lấy từ scope cha, không tự binding.
• Explicit binding: dùng
call, apply, bind để set this.
JSDebounce và Throttle là gì?• Debounce: Chỉ chạy function sau một khoảng thời gian khi không còn event nào xảy ra (VD: search input).
• Throttle: Giới hạn số lần function được gọi trong một khoảng thời gian (VD: scroll event).
JSsetTimeout(fn, 0) có thực sự chạy ngay lập tức không?• Không. Vì setTimeout được đưa vào Web APIs, sau đó callback được đẩy vào Callback Queue và phải đợi Call Stack rỗng mới chạy.
JSSự khác biệt giữa localStorage, sessionStorage và cookies?• localStorage: Lưu trữ vĩnh viễn, không bị mất khi reload.
•
sessionStorage: Chỉ tồn tại trong phiên làm việc của trình duyệt.
•
cookies: Dữ liệu nhỏ, có thể gửi lên server.
JSCác trạng thái của Promise?• pending: Đang chờ xử lý.
•
fulfilled: Thành công.
•
rejected: Thất bại.
JSSự khác biệt giữa == và ===?• ==: So sánh giá trị, có ép kiểu nếu cần. ("5" == 5 → true)
•
===: So sánh cả giá trị và kiểu dữ liệu. ("5" === 5 → false)
JSPromise và async/await khác nhau thế nào?Promise và async/await đều dùng để xử lý bất đồng bộ, nhưng có một số điểm khác biệt:
•
Promise:
◦ Cung cấp các phương thức
.then(), .catch(), .finally().
◦ Dùng nhiều
.then() lồng nhau có thể gây khó đọc (callback hell).
→ Giúp quản lý bất đồng bộ tốt hơn callback nhưng vẫn hơi rối.
•
async/await:
◦ Giúp code trông giống đồng bộ, dễ đọc hơn.
◦
await chỉ hoạt động trong hàm có async.
◦ Dễ xử lý lỗi hơn với
try...catch.
→ Giúp code dễ đọc, dễ debug hơn.
ReactJSVirtual DOM là gì? React hoạt động như thế nào?• Virtual DOM là bản sao của Real DOM, giúp React cập nhật UI một cách hiệu quả.
• Khi state thay đổi:
1. React tạo một Virtual DOM mới.
2. So sánh với Virtual DOM cũ (diffing).
3. Chỉ cập nhật những phần thay đổi trên Real DOM (reconciliation).
ReactJSSự khác biệt giữa Class Component và Functional Component?• Class Component:
◦ Dùng
this.state và this.setState().
◦ Dùng
componentDidMount, componentDidUpdate, v.v. để quản lý lifecycle.
•
Functional Component:
◦ Dùng
useState, useEffect thay cho state và lifecycle.
◦ Viết code ngắn gọn hơn, hiệu suất tốt hơn.
ReactJSState và Props khác nhau như thế nào?• State: Quản lý dữ liệu bên trong component, có thể thay đổi.
•
Props: Truyền dữ liệu từ component cha sang component con, không thể thay đổi (immutable).
ReactJSReact Hook là gì? Kể tên một số hooks phổ biến.• Hooks giúp Functional Component có thể sử dụng state và lifecycle.
• Một số hooks phổ biến:
◦
useState(): Quản lý state.
◦
useEffect(): Xử lý side effects.
◦
useContext(): Quản lý context.
◦
useRef(): Giữ tham chiếu không làm re-render.
◦
useReducer(): Quản lý state phức tạp.
ReactJSSự khác nhau giữa useEffect và useLayoutEffect?• useEffect: chạy sau khi render hoàn tất.
•
useLayoutEffect: chạy ngay sau khi DOM update nhưng trước khi browser vẽ giao diện.
ReactJSKhi nào nên dùng useMemo và useCallback?• useMemo: tránh tính toán lại giá trị nếu dependencies không đổi.
•
useCallback: tránh tạo lại function nếu dependencies không đổi (hữu ích khi truyền props xuống component con).
ReactJSContext API khác Redux như thế nào?• Context API: nhẹ, dùng để chia sẻ state đơn giản.
• Redux: mạnh hơn, quản lý state toàn cục với reducer, middleware, devtools.
ReactJSHigher Order Component (HOC) là gì?• Higher-Order Component (HOC) là một pattern trong React giúp tái sử dụng logic bằng cách bọc một component khác và mở rộng chức năng của nó.
ReactJSRe-render trong React xảy ra khi nào? Làm sao để tối ưu?• Khi state hoặc props thay đổi.
• Khi parent component re-render, child component cũng re-render theo (trừ khi dùng
React.memo).
•
Cách tối ưu:
◦ Dùng
React.memo để tránh re-render không cần thiết.
◦ Dùng
useCallback, useMemo để cache function/giá trị.
◦ Tránh cập nhật state trực tiếp trong
useEffect.
ReactJSSự khác biệt giữa Server Components và Client Components trong React?• Server Components: Chạy trên server, không tải JS xuống client, tốt cho SEO và performance.
•
Client Components: Chạy trên browser, có thể tương tác với event, state.
ReactJSTại sao key trong React nên là unique và không dùng index của array?• Nếu dùng index, khi danh sách thay đổi (thêm/xóa phần tử ở giữa), React có thể render sai component do index thay đổi.
• Key giúp React xác định chính xác component nào thay đổi
ReactJSReact Suspense và React Lazy là gì?• React Suspense: Tạo hiệu ứng loading khi tải component.
•
React Lazy: Chia nhỏ code để chỉ tải khi cần.
ReactJSCách tối ưu performance của React app khi có nhiều component con?• Code splitting (React.lazy + Suspense).
• Memoization (
React.memo, useMemo, useCallback).
• Batching updates với
useTransition hoặc useDeferredValue.
• Tránh re-render không cần thiết bằng kiểm tra dependency trong
useEffect.
ReactJSReact Portal là gì?Portal giúp render component ra ngoài root element.
ReactJSStrict Mode trong React là gì?Strict Mode giúp phát hiện các lỗi tiềm ẩn trong React, chỉ chạy trong dev mode.
VueJSSự khác biệt giữa Vue2 và Vue3?• Vue3 cải tiến hiệu suất, dùng Composition API, hỗ trợ TypeScript tốt hơn.
•
